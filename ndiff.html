Highlight discrepencies

I’ve added another metric to Laura’s columns. Defined as

\[\frac{\hat{N}_{mcds}-\hat{N}_{ddf}}{\hat{N}_{mcds}} \times 100\] This is depicted in the last column of the data frame. If the absolute of this value exceeds 10%, the row is highlighted.

library(knitr)
library(kableExtra)
j <- list.files("E:/mcds-compare", pattern="ROBJ$")
for (i in 1:length(j)) {
  fname <- paste0("E:/mcds-compare/", j[i])
  load(fname)
  pctdiff <- (model.compare$Nhat_MCDS - model.compare$Nhat_R)/model.compare$Nhat_MCDS * 100
  model.compare$Nhatdiff <- round(pctdiff, 2)
  setname <- unlist(strsplit(j[i], split = ".", fixed = TRUE))[1]
  print(kbl(model.compare, caption=setname)  %>%
          row_spec(which(abs(model.compare$Nhatdiff)>10), background="salmon"))
  cat("\n\n")
}

Obvious errors from highlights

There are a few data sets that have obvious problems:

  • dathr1
    • the hazard rate with adjustments disagree badly between optimisers.
  • ducknests
    • the likeilihoods and \(\hat{P}_a\) values for the half normal without adjustments is badly wrong. The likelihoods are also not the same as the likelihoods produced when the ducknest data set is analysed manually by DistWin.
  • duiker camera traps
    • uniform cosine just exceeds the 10% threshold, while the half normal Hermite adjustment models differ by 13%
  • LTExercise (simulated line transect data)
    • for both unifcos(2) and hrpoly(2) the R likelihood appears to be erroneous
  • Rocio data2
    • very small differences in likelihoods for hncos(2) and hrpoly(1) produce fairly large differences in abundance estimates
  • Savannah sparrow 80 and 81 (point transect data)
    • for 80, the troubled models both seem to have R likelihoods that are unreasonable, I think. For 81 unicos(2) shows problems with the R optimiser, while unicos(3) the problem lies with the mcds optimmiser
  • the 4 wren data sets
    • 5min
      • hnherm(1) problem might be with mcds, 15% discrepancy
    • cuecount (overdispersion)
      • four models with problems, worst problem being mcds optimiser with unicos(3)
    • line transect data shows no model disagreements (worst is hrpoly(2))
    • snapshot
      • hnherm(1) something badly wrong with the R optimiser for an otherwise well-behaved data set
